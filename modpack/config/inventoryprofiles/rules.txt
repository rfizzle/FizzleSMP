// Override IPN's default and creative_menu_order rules to avoid the
// native ::search_tab_index / creative-tab walk, which freezes clients
// on first use while building every creative tab.
// See: https://github.com/blackd/Inventory-Profiles/issues/440
//
// Both @default and @creative_menu_order are redirected to a cheap
// name-based sort; @default_components_rule is preserved so equipment
// with enchantments/NBT still clusters correctly.

@default
    ::custom_name
    ::display_name
    @default_components_rule

@creative_menu_order
    ::custom_name
    ::display_name
    @default_components_rule
